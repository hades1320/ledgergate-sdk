# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public
issue**.

Instead, report it privately by emailing: **security@yourprojectdomain.com**

Please include: - A clear description of the issue - Steps to reproduce
(if applicable) - Potential impact

---

## Scope

This project prioritizes security in the following areas: - Data privacy
and redaction - Authentication and authorization - API key handling -
Dependency safety

---

## Non-Goals

This project does **not**: - Handle or store private keys - Process
payments - Custody funds

Vulnerabilities related to wallet custody or transaction signing are out
of scope.

---

## Disclosure Process

1.  Issue is acknowledged within 72 hours
2.  Fix or mitigation is developed
3.  Coordinated disclosure after patch release

---

## Responsible Disclosure

We appreciate responsible disclosure and will credit reporters where
appropriate.
